## 2025-04-15 - Light Theme Implementation for Institutional Pages
**Learning:** When implementing theme toggles or section-specific themes in Tailwind CSS v4 using CSS variables, be careful to override all relevant CSS variables from the default/dark theme (like `--color-text-primary`, `--color-text-secondary`, and `--color-border-subtle`). Simply setting `color: #xxx` on the wrapper class is insufficient because child components (like `.text-lead`, `.interface-card`, or raw text elements using `color: var(--color-text-secondary)`) will inherit the original dark mode variable values, resulting in severe accessibility failures (e.g., light text on a light background).

**Action:** Updated `.theme-light` class in `globals.css` to redefine `--color-text-primary` (`#1F2937`), `--color-text-secondary` (`#6B7280`), and `--color-border-subtle` (`rgba(0, 0, 0, 0.12)`). Wrapped institutional pages (About, Pricing, How It Works, Documentation, Blog, Contact) with `<div className="theme-light">` to enforce proper contrast and styling.
