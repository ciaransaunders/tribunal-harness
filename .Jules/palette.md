## 2026-04-13 - Explicit Label Associations and Expandable Menu Controls
**Learning:** React `<label>` elements missing explicit implicit bindings (e.g., input not nested inside the label, and no `htmlFor` matching the input `id`) create significant barriers for screen readers, as form controls lose their programmatic context. Additionally, custom expandable menus and dropdowns (like the Trust dropdown and mobile hamburger menu) fail to convey their state and relationships to assistive technologies without proper `aria-controls` pointing to the target container's `id` and accurate `aria-expanded` attributes.
**Action:** Always ensure that form labels are explicitly bound using the `htmlFor` attribute that strictly matches the form control's `id`. For any interactive element that controls a collapsible/expandable region, enforce the implementation of `aria-controls` referencing the container's `id` alongside dynamic `aria-expanded` values reflecting the current state.
